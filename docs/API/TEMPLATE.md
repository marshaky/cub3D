# API Documentation Template

Use this exact structure for all API function pages.

## Template

```md
<a name="top"></a>
# [⬅️ Back to MODULE Documentation](./README.md)

# function_name

Declaration:
```c
return_type function_name(args);
```

Summary:

One short paragraph explaining what the function does.

Parameters:

`arg_name` - short description.

Return value:

`type/value` and meaning. If void, write: "No return value."

Flow:

1. Step one.
2. Step two.
3. Step three.

Notes:

Optional implementation details, constraints, caveats.

---

## Style rules

- Keep docs in English.
- Keep sections short and practical.
- Prefer behavior + reason, not only code repetition.
- Use consistent terms: "ray", "wall column", "camera plane", "map grid".
